# Service pattern

Services hold business logic and implement interfaces declared in
`domain/port.go`. The interface uses **plain Go types** (strings, ints,
domain structs) — never `api.*` generated types. That keeps `domain/`
import-clean (zero framework/adapter imports).

The handler's job is to decode `api.CreateFooRequest` and pass plain args
to the service. See `patterns/handler.md`.

## Skeleton

```go
// In domain/port.go (plain types only — no api imports):
type OrderService interface {
    Create(ctx context.Context, userID string, totalCents int) (*Order, error)
    Get(ctx context.Context, userID, id string) (*Order, error)
}

// In service/orders.go:
type OrdersService struct {
    orders domain.OrderRepository
    users  domain.UserRepository
    logger *slog.Logger
}

func NewOrdersService(orders domain.OrderRepository, users domain.UserRepository) *OrdersService {
    return &OrdersService{orders: orders, users: users, logger: slog.Default()}
}

func (s *OrdersService) Create(ctx context.Context, userID string, totalCents int) (*domain.Order, error) {
    if totalCents <= 0 {
        return nil, domain.ErrInvalidInput
    }

    user, err := s.users.FindByID(ctx, userID)
    if err != nil {
        return nil, fmt.Errorf("loading user: %w", err)
    }
    if !user.CanPlaceOrder() {
        return nil, domain.ErrOrderForbidden
    }

    order := &domain.Order{UserID: userID, TotalCents: totalCents}
    if err := s.orders.Save(ctx, order); err != nil {
        return nil, fmt.Errorf("saving order: %w", err)
    }
    return order, nil
}
```

## Rules

- **Interfaces in `domain/port.go` use plain types only** — no `api.*`, no
  HTTP types, no SQL. The handler adapts `api.CreateOrderRequest` into
  plain args before calling.
- Accept interfaces (`domain.OrderRepository`), return structs (`*domain.Order`).
- `context.Context` is always the first parameter.
- Wrap every error with meaningful context — `fmt.Errorf("operation: %w", err)`.
- Return domain error sentinels for expected cases. **Reuse
  `domain.ErrInvalidInput` for generic 400s** (empty required string,
  out-of-range number). Create a new sentinel only when the client needs
  to distinguish the case programmatically (e.g., `NOTE_LIMIT_EXCEEDED`).
  If you add a sentinel you MUST also update `respond.go`'s `mapError`,
  `error_envelope_test.go`, and every `openapi.yaml` endpoint that can
  return it.
- Let infrastructure errors bubble up wrapped.
- Use `slog` for logging. Redact PII.

## IDs

Primary keys are UUIDs generated by Postgres via `DEFAULT gen_random_uuid()`
(in both the migration and the GORM tag). Services **never** call
`uuid.NewString()`. The repo copies the DB-assigned ID back into the
domain entity after `INSERT`. See `adapter/persistence/user_repo.go`.

## Testing

- Mock repositories via hand-written stubs in the test file (follow
  `service/auth_test.go`).
- Table-driven tests. Name: `TestOrdersService_Create_UserCannotOrder_ReturnsForbidden`.
- Assert on behavior (return value, repository calls), not implementation detail.
